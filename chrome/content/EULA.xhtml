PRIVACY IMPLICATIONS OF THIS PROTOTYPE

In this version of the Synchotron 9000+TM we unleash the awesome. However, to do so, we must take some liberties with your private data. This document outlines what data this service collects, how we protect it, and how it will be used.

Data being transmitted currently

  • all bookmarks (starred, tagged or in the heirarchy)
  • microsummaries
  • live bookmarks
  • number of bookmarks
  • user-agent strings, IP addresses, etc. transmitted as part of an authenticated session

Data protection

  • All processing happens client-side and the server is only passive storage
  • All of your bookmarks data is trasmitted over SSL to https://services.mozilla.com/ and is stored on a WebDAV service in your own user directory. Only you and the Mozilla IT team have access to this user directory.
  • This version does not encrypt your bookmark data. Future versions will include client-side on-by-default AES encryption as we do not need or want to access anyone's bookmark data.
  • Regardless, we will not farm, mine or otherwise examine your personal bookmarks without your permission.

How do we use any information collected

  • Number of bookmarks on average and in aggregate, as well as Web server statistics will be collected and analysed in order to detect bottlenecks, better understand the scalability of the service, etc.

For more information on this prototype, please visit: https://intranet.mozilla.org/Labs/Synchotron.
